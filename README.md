# Minecraft Clone

A voxel-based game engine built with C++, Qt6, and OpenGL, inspired by Minecraft.

## Features

- **Voxel Rendering**: Efficient chunk-based terrain rendering with face culling
- **Block System**: Multiple block types with texture atlas support
- **Camera System**: First-person camera with WASD movement and mouse look
- **Multi-threaded Logger**: Async logging system with color-coded console output
- **Procedural Terrain**: Simple terrain generation using height maps

## Project Structure

```
Minecraft/
├── src/
│   ├── Core/           # Core systems (Time, Input)
│   ├── Render/         # Rendering (Shader, Texture, Camera)
│   ├── World/          # World generation (Block, Chunk)
│   ├── UI/             # Qt UI widgets (GameWidget)
│   └── Utils/          # Utilities (Logger, GLM, stb_image)
├── bin/
│   └── Resource/       # Game resources (shaders, textures)
├── 3rdparty/           # Third-party libraries
│   ├── qt6/            # Qt6 framework
│   ├── GLEW/           # OpenGL Extension Wrangler
│   └── Assimp/         # Asset importer (future use)
└── build/              # Build output directory
```

## Dependencies

## Building

### Prerequisites

- Visual Studio 2022 (MSVC v143)
- CMake 3.16+
- Windows SDK 10.0.26100.0+

### Build Steps

1. Clone the repository
2. Navigate to the build directory:
   ```bash
   cd build
   ```
3. Run the build script:
   ```bash
   # Debug build
   .\win_build.bat debug
   
   # Release build
   .\win_build.bat release
   ```

### Build Output

- **Compilation output**: `build/MinecraftSymbols/win64/bin/Debug/Minecraft.exe`
- **Installation output**: `build/Minecraft/win64/bin/Minecraft.exe`

Both executables are ready to run with all dependencies and resources copied.

## Controls

- **W/A/S/D**: Move forward/left/backward/right
- **Space**: Move up
- **Shift**: Move down
- **Mouse**: Look around
- **ESC**: Exit game

## Technical Details

### Rendering Pipeline

1. **Chunk Generation**: 16x256x16 blocks per chunk
2. **Mesh Building**: Only visible faces are rendered (face culling)
3. **Texture Atlas**: 16x6 grid texture atlas for block textures
4. **Lighting**: Per-face ambient occlusion (top=1.0, sides=0.8, bottom=0.5)

### Block Types

- Air (transparent)
- Grass (textured top/side/bottom)
- Dirt
- Stone
- Wood (textured top/side/bottom)
- Leaves (transparent)
- Sand
- Water (transparent)
- Glass (transparent)

### Qt MOC Integration

This project uses manual MOC (Meta-Object Compiler) generation:
- MOC files are generated by `win_build.bat` before compilation
- Generated files are placed in `build/MinecraftSymbols/win64/moc_generated/`
- To add new Q_OBJECT classes, update the MOC generation section in `win_build.bat`

## Logger System

The project includes a multi-threaded async logger:
- Thread-safe logging with mutex protection
- Color-coded console output (Info=Green, Warning=Yellow, Error=Red, Fatal=Magenta)
- File logging to `minecraft.log`
- Thread ID tracking for debugging

Usage:
```cpp
LOG_INFO("Information message");
LOG_WARNING("Warning message");
LOG_ERROR("Error message");
LOG_FATAL("Fatal error message");
```

## Resource Files

Resources are located in `bin/Resource/`:
- **Shaders**: `Resource/Shader/basic.vert`, `basic.frag`
- **Textures**: `Resource/Texture/default_texture.png` (16x6 block atlas), `sun.png`

The build system automatically copies resources to the output directory.
